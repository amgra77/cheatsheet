Doctrine

Command Description
php app/console doctrine:generate:entity (NO params) Helper to create new table, will create the Entity.php, later you have to run command to update database which actually will create the new table phisically
php app/console doctrine:generate:entities ManwinAFBundle:Publisher Will create getters and setters, it is safe to run it as many times as possible because will not overwrite your current methods if there is any
php app/console doctrine:schema:update --dump-sql will SHOW any changes made on the annotations that need to be reflected into the real db
php app/console doctrine:schema:update --force will MAKE (alter table) any changes made on the annotations that need to be reflected into the real db
php app/console doctrine:database:create Assistant to create a database
php app/console doctrine:database:drop --force will DROP completely the dabase
WARNING
php app/console doctrine:mapping:convert xml ./src/Manwin/AFBundle/Resources/config/doctrine/metadata/orm NO_MORE_PARAMS Will read the entity.php and create XML files under folder specified, no writing on Entity or Database
php app/console doctrine:mapping:convert xml --from-database ./src/Manwin/AFBundle/Resources/config/doctrine/metadata/orm Will read the database and create XML files under folder specified, if the file.xml already exists it will NOT be overwritten but if it exists will show error, better to specify a temp folder or use filter option (below explained), so far no writing on Entity folder yet
php app/console doctrine:mapping:convert xml --from-database --filter="Spots2" ./src/Manwin/AFBundle/Resources/config/doctrine/metadata/orm Will read the database and create XML files under folder specified, --filter will FILTER only X table, so far no writing on Entity folder yet
php app/console doctrine:mapping:convert xml --from-database
--force
./src/Manwin/AFBundle/Resources/config/doctrine/metadata/orm		 Will read the database and create XML files under folder specified, --force will OVERWRITE the existing files.xml, so far no writing on Entity folder yet
BE careful, --filter="Spot" will match Spot*
php app/console doctrine:mapping:import ManwinAFBundle annotation --filter="Spot" This will
OVERWRITE
the entity.php files reading all tables from the DB, you can use the filter option to make only xxx table
